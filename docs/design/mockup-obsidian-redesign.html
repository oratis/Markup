# 广告投放数据直拉计划 — 绕过 Adjust, Luddi DB 成 SSoT

触发：2026-05-22 排查 /console/analytics Tab 1 / Tab 2 cost 数据时发现 Adjust → Luddi cost 通道全断。TikTok 100% spend 缺失（$0），Google Ads 通但混了其他 HakkoAI 账号（虚高 ~30%）。

决策：彻底绕过 Adjust 拉 cost，直接调 Google Ads API + TikTok Marketing API。Adjust 保留做 cross-network install attribution（不能丢），但所有 cost 数据改由我们自己存的 AdSpendDaily 表为准。

状态（2026-05-22）：Auth + 数据探索完成。PR 1-4 待执行。

# 1. 背景与现状

1.1 Adjust ReportSnapshot 当前数据状态（直连 prod DB 实测）

TikTok SAN     | 1148 rows | 218 installs | $0.00    | 0 rows with cost  ✗
Google Ads ACI | 228 rows  | 273 installs | $1,070.94 | 119 rows with cost ⚠ 混了其他 HakkoAI 账号
Organic        | 123 rows  | 317 installs | $0.00    | (by definition)

1.2 同期直拉 API 实测对照

数据源 TikTok (5/14-5/22) Google Ads (5/14-5/21)
Adjust ReportSnapshot（Luddi 现状） $0 ✗ $1,019.83 (含 Hakko 其他账号)
直拉 Marketing API（本计划） $1,301.35 ✓ 4 advertiser $786.66 ✓ 3 customer, Luddi-only

# 2. 设计

新建表 [[AdSpendDaily]]，每天通过 Cron job 把 Google Ads + TikTok 数据归一化写入；console analytics 查询时直接 join 此表，不再调用 Adjust ReportSnapshot 的 cost 字段。

Note
Adjust 的 attribution data 仍然保留 — 我们只切断 cost 通路。详见 [[database-schema]] 第 §3 节。
