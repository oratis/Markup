# markup — 项目总览

## 一句话定位

> **markup** 是一个为 macOS 打造的高性能 Markdown 编辑器，复刻 Typora 的所见即所得体验，并补齐它的性能软肋与现代化能力。

## 命名

- 应用名：**markup**（小写）
- Bundle ID：`com.appkon.markup`
- 仓库名：`MarkUp`（保持当前目录名）
- 显示名：「Markup」

## 目标用户

第一阶段：**项目所有者本人**。
- 在 Mac 上写技术笔记 / 设计文档 / 个人知识库
- 已用过 Typora / Obsidian / VSCode markdown
- 对性能敏感，文件可能 MB 级或文档很多

未来阶段（V1+）：愿意付出"开源工具学习成本"换取性能与本地优先的开发者群体。

## 与 Typora 的关系

- **不是** fork，**不是** 二进制改造，**不是** 商业克隆
- 是**功能上的 spiritual successor**：保留 Typora 最被喜爱的"单视图 WYSIWYG"心流体验
- 用现代技术栈（Tauri / ProseMirror / Rust）重做，目标是让"Typora 风格 + 极致性能 + 大 vault 管理"三者同时成立

## 核心承诺

| # | 承诺 | 度量方式 |
|---|------|----------|
| 1 | **Typora 心流不丢**：单视图 WYSIWYG、Focus、Typewriter 完整可用 | 用户主观验收 |
| 2 | **打开大文件不卡** | 5MB 文档 < 500ms 打开、输入延迟 < 16ms |
| 3 | **管理大 vault 不慢** | 1 万文件索引 < 5s、全文搜索单位毫秒 |
| 4 | **启动毫秒级** | 冷启动 < 1.5s |
| 5 | **小巧** | 安装包 < 20MB，常驻内存 < 300MB |
| 6 | **Mac 原生体感** | 菜单栏、快捷键、文件关联、暗黑模式跟随系统 |

## 显式 Non-Goals（V1 不做）

- ❌ Windows / Linux 版（专注 Mac 把体验做透）
- ❌ iOS / iPad 版
- ❌ 云同步 / 账号系统（本地优先；用户可自己用 iCloud Drive / Dropbox）
- ❌ 协作编辑 / CRDT
- ❌ 笔记图谱 / 双向链接（V1.x 再加，先把基础做扎实）
- ❌ 数据库式属性（Notion 风格）
- ❌ AI 集成（先把编辑器做好，AI 是 V2 话题）
- ❌ 插件市场（V1 先内置插件，开放 API 是 V2）

## 哲学

1. **本地优先**：纯文件 + 纯文件夹，没有专有数据库。用户的 markdown 用任何编辑器都能打开
2. **性能即特性**：每个加进来的功能都要过性能检查
3. **文件即真相**：编辑器状态可重建，磁盘上的 .md 是唯一权威
4. **可逆离开**：用户随时能停用 markup，文件不会留下任何专有元数据污染
