# ADR-003: 产品定位、1.0 范围与分发

- **状态**: 已批准（2026-06-10）
- **日期**: 2026-06-10
- **依据**: [REVIEW-2026-06.md](../REVIEW-2026-06.md)、[PRODUCT-DIRECTION.md](../PRODUCT-DIRECTION.md)
- **关系**: 延续 [ADR-002](./ADR-002-distribution.md)（分发档位）；澄清 [RELEASE-PLAN](../RELEASE-PLAN.md) 中"MAS-vs-direct 硬阻塞"

## 上下文

2026-06 全项目审查后，工程上距 1.0 几乎无硬缺口，但存在两个待定问题：

1. **定位分散**：产品同时对标 Typora（reader-first 单文件）、Obsidian（知识库）、GitHub 文档阅读器（iOS 主打），叙事与精力被稀释。
2. **1.0 分发**：RELEASE-PLAN 长期把"上 Mac App Store 还是只做直分发"列为唯一硬阻塞，迟迟未拍板。

需要一次决策把方向定下来，避免继续在三个身份间摇摆。

## 决策

### 1. 产品聚焦"阅读器优先 × 知识库"象限

竞品分布：Typora/iA/MacDown 在"编辑器优先×单文件"，Obsidian 在"编辑器优先×知识库"。Markup 是唯一**把渲染当产品**的，独占"阅读器优先×知识库"空白象限。叙事与路线图围绕**阅读**与 **GitHub** 排序；Canvas/图谱作为知识库能力的支撑，不作为对标 Obsidian 的主战场。

- **不打**：社区插件生态、跨设备实时同步（Obsidian 护城河，消耗战）。
- **要打**：reader-first 体验、GitHub 文档原生阅读与轻量协作、Mac↔iOS 一致性。

### 2. 立"GitHub 文档原生阅读器（Mac + iOS）"为主打叙事

iOS 已验证"打开任意 GitHub 仓库当 vault 离线阅读"的价值；桌面端补齐并双向化（编辑副本 → PR/fork）。这是当前最被低估、最可防守的方向。

### 3. 1.0 走直分发优先，MAS 推迟为 1.0 后增量

延续 ADR-002 的"档位 1"：1.0 以已签名公证 + 自动更新的直分发 DMG 发布，**不阻塞于 App Store**。MAS 代码已就绪（沙箱、安全作用域书签、隐私政策），作为 1.0 后触达增量再启动。据此，RELEASE-PLAN 中"唯一硬阻塞=MAS 决策"的措辞作废。

## 后果

- **正面**：叙事清晰可传播；开发精力集中在差异化（阅读 + GitHub）；1.0 不再被账号/审核流程卡住。
- **代价**：明确放弃插件生态与实时同步两类需求的用户；GitHub 双向化需新增 OAuth 写 scope、PR/fork 逻辑（B 层工作）。
- **可逆性**：定位是叙事与排序选择，未来可调整；分发决策可随时叠加 MAS，无技术债。

## 用户确认结果（2026-06-10）

用户审阅本次审查后指示"按推荐顺序全部推进"，即采纳上述三项决策，并授权按 PRODUCT-DIRECTION 的 A 层路线图开始实施。
